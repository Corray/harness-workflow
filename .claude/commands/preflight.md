# /preflight

提交前全面检查。

## 执行流程

### 1. 编译
运行编译命令（如 `mvn compile` / `npm run build`），确认无错误。

### 2. 运行测试
运行所有测试（包括本次新增的）。

### 3. 覆盖率检查
对照 `.claude/knowledge/testing/standards.md` 中的覆盖率要求：
- 单元测试覆盖率
- 分支覆盖率

### 4. 红线扫描
对变更文件执行 `.claude/knowledge/red-lines.md` 的全量扫描。

### 5. 设计对齐
对比设计文档，确认没有实现偏差。

### 6. 输出报告

```
## Preflight 报告

✅ 编译：通过
✅ 测试：45/45 通过
✅ 覆盖率：82%（要求 ≥80%）
✅ 红线扫描：通过
⚠️ 设计对齐：1 处偏差 — {描述}

### 结论
- 全部通过 → 可以提 PR
- 有警告 → 建议先解决再提 PR
```

### 7. 生成 PR 描述（如全部通过）

```
## 变更概述
{根据 journal 和设计生成}

## 变更范围
- {文件列表}

## 测试
- {测试用例概述}

## 可追溯
- 共识文档：{版本}
- 迭代共识：{路径}
- 设计：{路径}
```

### 8. 人工审批提示

> 所有检查通过。
> 请最终确认后执行 git commit 并推送 PR。
> **不会自动 commit 或 push。**
