# /update-rules

根据 eval 的根因分析，路由修改到对应规则文件。

## 参数

- `eval-report`（必填）：最近一次 /eval 的输出，或指向报告文件的路径

## 执行流程

### 1. 读取 eval 报告
从参数或 `eval/reports/` 最新文件中读取根因分析。

### 2. 按文件分组修改建议

根据根因映射表：

| 根因类型 | 目标文件 |
|----------|----------|
| 流程编排问题 | .claude/commands/*.md |
| 内容规则问题 | instructions.md |
| 角色混淆 | CLAUDE.md |
| 知识缺失 | knowledge/*.md（按主题选择） |
| 评估标准缺陷 | eval/dimensions.md |

### 3. 逐文件生成修改方案

每个文件的修改方案包含：
- 当前版本的问题位置（引用行号或段落）
- 建议的修改内容
- 修改理由（对应 eval 报告中的扣分项）

### 4. 人工按文件确认

**支持部分确认** — 每个文件独立确认，用户可以：
- 全部接受
- 部分接受（指定哪些修改执行）
- 全部拒绝

提示格式：
> 文件：instructions.md
>
> 修改 1：新增禁用词列表
> 理由：B2 扣 3 分 - 出现"等"模糊表述
> 预览：
> ```
> + ### 禁用词黑名单
> + - "等" / "类似" / "相关" — 必须穷举
> ```
>
> 是否应用？[全部 / 选择 / 跳过]

### 5. 执行修改

用户确认的修改逐一应用到文件。

### 6. Git commit

```
git commit -m "rules: fix {维度}{编号} ({旧分}→{新分})"
```

例：`git commit -m "rules: fix E3+B1 (38→43)"`

### 7. 建议回归测试

提示：
> 规则已更新。建议执行 /eval {case} 进行回归测试，
> 验证改善且无新的回退。
